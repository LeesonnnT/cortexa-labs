# Cortexa Labs

Cortexa is a context-first CLI for AI-assisted engineering. It turns an existing project into structured, task-sized context so AI coding tools can read less, stay inside clearer boundaries, and produce more stable changes.

The core product promise is simple:

> Given a real engineering task, `ctx pack` returns the smallest useful Context Packet with evidence, reading order, risk boundaries, and validation hints.

## Quick Start

Inside an existing npm project:

```bash
npm create cortexa@latest -- --yes --task "fix login token expiration"
```

After Cortexa is installed, the everyday path is one command:

```bash
npx --no-install ctx go --explain "fix login token expiration"
```

Choose a template and editor integrations explicitly:

```bash
npm create cortexa@latest -- --template frontend --editors codex,cursor --task "fix login token expiration"
```

## Common Commands

```bash
npx --no-install ctx setup --template auto --editors codex
npx --no-install ctx discover
npx --no-install ctx analyze
npx --no-install ctx audit
npx --no-install ctx go --explain "<task>"
npx --no-install ctx pack --explain "<task>"
npx --no-install ctx update
npx --no-install ctx teardown
```

`ctx go` creates or refreshes the local `.cortexa/` context assets and then returns a Context Packet for the task. Use `ctx pack --explain` when you only want to compile context without setup/update side effects. `ctx audit` checks whether generated snapshots, manifests, and core assets are healthy.

## Generated Assets

Setup creates a `.cortexa/` directory in the target project:

```txt
.cortexa/
├─ agents/          # role profiles for AI collaboration
├─ skills/          # engineering skills and execution steps
├─ specs/           # requirements / design / tasks conventions
├─ contexts/        # Context Packet schema and definitions
├─ adapters/        # adapter discovery snapshots
├─ graphs/          # repo graph snapshots
├─ runtime/         # sessions and cache lifecycle
├─ ownership/       # package and feature ownership map
├─ multi-agent/     # collaboration protocol and handoff schema
├─ workflows/       # Context Flow conventions
├─ reports/         # generated by analyze/audit/review commands
├─ context-manifest.json
├─ integrations.json
├─ project-kit.json
├─ starter-kit.json
└─ workspace.json
```

Human-owned files are created only when missing. Machine-owned snapshots can be refreshed with `ctx update`. Hybrid assets refresh only Cortexa-managed blocks and preserve team edits.

## Development

This repository contains the npm initializer and the local CLI:

- `apps/create-cortexa`: package behind `npm create cortexa@latest`
- `apps/cli`: package behind `@cortexa-labs/cli` and the `ctx` command
- `examples/minimal`: smoke-test project for the setup/pack/audit flow

Run local checks:

```bash
npm test
npm run check
```

`npm run check` runs release-oriented checks: package metadata, executable entrypoints, documentation, CLI smoke tests, unit tests, and an example lifecycle.

If npm is not available on your machine but Node is, you can run the release script directly:

```bash
node scripts/check-release.js
```

## CI and Release Gate

GitHub Actions runs:

```bash
npm ci
npm test
npm run check:release
npm pack --workspace apps/cli --dry-run
npm pack --workspace apps/create-cortexa --dry-run
```

Before publishing, both packages should pass the same release gate locally or in CI.
